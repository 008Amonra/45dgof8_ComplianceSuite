Our Security Standards

Conservative claims • Human review • Source transparency

Editorial Standards

  • Source minimum: At least two reputable sources per post; all links visible.
  • Dates & timezone: Absolute dates with Europe/Zurich timezone.
  • Clarity: TL;DR + “What happened / Why it matters / Actions” in every post.
  • Tone: Conservative language, no hype or speculation.
  • Review: AI‑assisted drafts are human‑reviewed before publishing (unless auto‑publish is enabled by you).
  • Corrections: Updates and errata are timestamped and appended.

Operational Standards

  • Automations: Continuous monitoring via Zapier (cloud) or n8n (self‑hosted/cloud).
  • Access: Least‑privilege; only required scopes are granted.
  • Secrets: API keys and tokens stored in platform vaults; never committed to code.
  • Transport: TLS for data in transit; encrypted storage for credentials.
  • Logging: Minimal, purpose‑limited logs with retention controls.
  • Change control: Workflow changes are tested in staging before production.
